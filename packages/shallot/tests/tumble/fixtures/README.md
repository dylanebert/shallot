# tumble fixtures — C-generated bit-exact contract

The 53 `*.json` files here are the tumble engine's regression truth: per-step FNV-1a world-state hashes
(plus periodic body-state dumps) generated by the Box3D C reference. `src/standard/tumble/engine/step.fixture.ts`
rebuilds each scene through the engine's public API, steps it, and asserts its hash equals the fixture's at
every step — at single-thread and every thread count (thread-count independence is by construction, so the
same hashes gate the MT kernel unchanged).

These are committed, not gitignored: they make the bit-exact contract self-contained, runnable anywhere with
no C toolchain or sibling checkout. They sit under `tests/` (outside `files: ["src"]`), so they never ship in
the npm package.

## Provenance

- **Box3D pin:** `29bf523` (box3d `v0.1.0` + the Fixes 01–02 batches), on the reference's `harness` fork branch.
- **Generator config:** built `BOX3D_DISABLE_SIMD=ON`, `BOX3D_FORCE_OVERFLOW=OFF` — the colored solver (graph
  coloring + the wide 4-lane convex path + serial mesh/overflow spill), which the engine mirrors per-lane.
  DISABLE_SIMD's scalar `FloatW` is bit-identical per lane to the SIMD build, so these fixtures pin the shipping
  wide-simd wasm path too.
- **Generator source:** `reference/box3d/fixtures/gen.c` (the `fixture_gen` target).

## Minting new fixtures (deliberate upstream sync only)

Default is freeze — the pin's whole value is the bit-exact contract; re-validating against a moving upstream is
pure cost. When a box3d coherence boundary earns a sync, regenerate with the C reference as the mint instrument:

```
bun run gen-tumble-fixtures          # scene fixtures (this dir)
bun run gen-tumble-gold <name>       # a committed gold vector (src/standard/tumble/engine/<name>.gold.json)
```

Both require `reference/box3d` checked out beside the shallot repo (the kex workspace layout: `kex/reference/box3d`,
sibling of `kex/shallot`) on the `harness` branch, plus cmake and a C toolchain. Absent it, the scripts error
honestly. Never hand-edit a fixture: it is C-generated truth, and a mismatch is an engine bug, not a fixture to
adjust.
