# Physics (AVBD Solver)

Reference: `docs/standard/physics.md` for concept, usage patterns, and API surface.

GPU-driven rigid body physics using Augmented Vertex Block Descent (Giles, Diaz, Yuksel â€” SIGGRAPH 2025).

## Authority

The AVBD paper (Giles, Diaz, Yuksel â€” SIGGRAPH 2025) is primary. Chris Giles' 3D reference demo is authoritative for constraint math; the 2D demo (savant117/avbd-demo2d) is reference for constraint math only.

Parameters: iterations=4, alpha=0.99, betaLin=100000, betaAng=100, gamma=0.999, gravity=-10. PENALTY_MAX=1e10, COLLISION_MARGIN=0.01. Jones-Plassmann graph coloring (priority-based deferral, 16 rounds), MAX_COLORS=12. Uncolored stragglers fall back to color 0 (Jacobi) â€” `sortBodiesByColor` redirects any body with assigned color â‰Ą MAX_COLORS to color 0. Primal writes directly to `bodies[idx]` per color â€” graph coloring guarantees no two threads in the same color touch the same body, and across colors the compute-pass barrier serializes the Gauss-Seidel updates. All colors dispatched every iteration (empty colors dispatch 0 workgroups, fast-pathed by driver).

**Primal reads neighbors from a column buffer, not AoS bodies.** `bodyCols` (binding 10) packs pos+radius / initial+colliderType / cumAng / quat / initialQuat into 5 `vec4f` columns Ă— capacity entries (80B/body vs 208B AoS). `syncBodyCols` mirrors AoS â†’ columns once per substep, right after `sortBodiesByColor`. Primal's contact path reads neighbor data via explicit column indices (`bodyCols[col * params.capacity + idx]`) and writes the body it owns to BOTH AoS and columns. Helper struct loads (`var body = bodies[idx]`) don't elide unused fields, so the inner loop must use explicit `vec4f` reads â€” see `contactArmFields` in `solver.wgsl.ts`. Other passes (warmstart, dual, narrowphase, character) keep AoS reads; only primal touches `bodyCols`.

## Collision filtering

`Body.group` (u32, default 0). Group 0 collides with everything. Same non-zero group = pair skipped in broadphase. GPU Body struct field `collisionGroup` at offset 48 (struct padded to 52 floats / 208 bytes for vec4f alignment). Broadphase check in `testBinaryChild` after `isJointed`, before `emitPair`.

## Collision detection

**Per-type indirect dispatch.** Broadphase classifies pairs by shape type during emission and scatters them into per-type sections of the pair buffer using `SS_PAIR_TYPE_BASE` atomic counters (10 types: box-box, sphere-box, capsule-box, sphere-sphere, capsule-sphere, capsule-capsule, hull-box, hull-sphere, hull-capsule, hull-hull). Each type has its own pipeline containing only its detection function, dispatched indirectly via `prepareIndirect`. Zero warp divergence â€” every thread in a workgroup runs the same collision algorithm. Adding a new shape type means adding one pipeline entry in `narrowphaseWGSL(pairType)`.

**4 collider types:** box (0), sphere (1), capsule (2), hull (3). Broadphase LUT is 4x4 (16 entries). Hull bodies store a GPU hull index in `body.hullId`. Hull vertex/face/edge data packed into a single `hullData` buffer (bind group 1, binding 1) with offset arithmetic.

Box-box uses SAT (15 axes). Sphere-sphere uses center-to-center distance (single contact). Sphere-box uses Ericson OBB clamping (single contact). Capsule-sphere uses closest-point-on-segment (single contact). Capsule-capsule uses closest-points-between-segments (single contact, ported from AVBD reference). Capsule-box tests both segment endpoints as sphere-box contacts (up to 2 contacts per pair, following Bullet's compound capsule pattern). Hull-box uses SAT with hull face normals + 3 box axes + edge cross products, face clipping. Hull-sphere uses closest-feature test (face/edge/vertex). Hull-capsule decomposes to two hull-sphere endpoint tests. Hull-hull uses full SAT with face normals from both hulls + edge-edge cross products. Feature keys: sphere-sphere `3u<<24`, sphere-box `4u<<24`, capsule-sphere `5u<<24`, capsule-capsule `6u<<24`, capsule-box `(7u<<24)|endpoint`, hull-sphere `8u<<24`, hull-capsule `(9u<<24)|endpoint`, hull-box `(10u<<24)|slot`, hull-hull `(11u<<24)|slot`. Edge/face bias: 0.95/0.01 â€” prefer face contacts unless edge separation is significantly deeper.

GJK/EPA and SDF were both tried and rejected â€” GJK has unpredictable iteration count and branching (GPU-hostile), SDF had accuracy issues. SAT has fixed iteration count per shape pair, no branching, parallelizes cleanly. Sphere/capsule use closest-point geometry (not SAT) but produce the same constraint output format.

**CPU SAT reference** (`sat.ts`). Pure CPU hull-hull SAT validated against Bullet3 gold vectors (`tests/sat-gold-vectors.json`, generated by an out-of-tree Bullet3 harness). Always uses face clipping for contact generation regardless of whether the separating axis came from a face normal or edge cross product (matching Bullet3). Hull-sphere uses closest-feature test, hull-capsule decomposes to two hull-sphere tests. Ground truth for GPU SAT.

**Body pair ordering:** Broadphase normalizes pairs during classification. Same-type pairs use `(min_idx, max_idx)` for deterministic SAT reference face selection. Mixed-type pairs put the higher-type body first (hull > capsule > sphere > box), matching each detection function's argument convention. Changing this ordering breaks the Sutherland-Hodgman clip symmetry and causes spurious rotation.

**Hull GPU data:** `packHullsForGPU()` in `hull.ts` consolidates all registered hulls into a single buffer. Layout: metadata region (12 u32 per hull: offsets + counts for vertices, faces, edges, plus 3 inverse-extent floats for runtime scale), then vertex region (vec4f per vertex), face region (plane equation + index metadata per face), face index region, edge region. Runtime scale: `hullScale(hm, body.halfExtents)` = `halfExtents * invExtent`, applied to vertices, face normals (inverse-transpose), face plane distances, and edge directions. Bounded iteration constants: MAX_HULL_VERTS=64, MAX_HULL_FACES=32, MAX_HULL_EDGES=48, MAX_FACE_VERTS=16. `contactArm` for hulls uses the box path (full body-local rotation). Inertia uses box approximation with hull AABB extents. Hull-sphere and hull-capsule use slot-based feature keys (not face indices) for warmstart stability across reference face changes.

## Constraint evaluation is type-specific

AVBD uses one augmented Lagrangian for all types, but evaluation differs per type. Don't collapse into shared code paths without verifying the reference does the same.

| Type | C evaluation | J evaluation |
|------|-------------|-------------|
| Contact | Taylor expansion with `contactArm` â€” shape-aware arms recomputed each iteration. Spheres: `Â±normal*radius` (quat-independent). Capsules: segment offset rotated by current quat + fixed radial along normal. Boxes: full body-local rotation | Same `contactArm` for Jacobians. Fresh arms every iteration, matching AVBD reference's updated-Jacobian approach |
| Kinematic | Same as Contact but alpha=0 (full correction on first frame). Auto-detected in narrowphase when either body has massâ‰¤0 and moved flag set. Tangent C_init biased by kinematic displacement (`body.vel`, stored by `syncTransforms` for `Move` bodies / `characterApply` for characters) so friction drags dynamic bodies along with moving platforms. Normal C_init unmodified (already correct from post-move geometry). Kinematic bodies without `Move` component are treated as teleports (vel=0, moved=0) | Same as Contact |
| Ball joint (hard) | Re-evaluated each iteration | Recomputed from current rotation |
| Ball joint (finite stiffness) | Re-evaluated, no stabilization | Recomputed from current rotation |
| Spring | Re-evaluated (distance - restLength) | Recomputed (normal updates) |

Hard constraints (contacts, hard ball joints with stiffness=0/default) use augmented Lagrangian (penalty + lambda). Soft constraints (springs, finite-stiffness ball joints) use penalty only â€” lambda always 0, penalty capped at stiffness. Finite-stiffness ball joints match the reference's `Joint` with finite `stiffnessLin` (used for grab interaction). The stiffness threshold is `>= 1e30` (hard) vs `< 1e30` (soft).

## Contact readback

GPU-side `emitContacts` pass runs after writeback. Iterates constraints, emits new contacts (warmstart miss, `isNew == 1`) above an impulse threshold. `GPUConstraint.isNew` (offset formerly `_pad0`) is set in `pushConstraintWithWarmstart` â€” 1 when featureKey is FEATURE_KEY_NONE (no warmstart hit). Output: flat u32 buffer with CONTACT_STRIDE=9 per contact (bodyA, bodyB, pos.xyz, normal.xyz, impulse as bitcast f32). Own bind group layout (4 bindings: bodies read-only, constraints read-only, solverState read-write, contacts storage). Counter slots `SS_CONTACT_COUNT` and `SS_CONTACT_OVERFLOW` in solverState, cleared with other counters. MAX_CONTACTS=128.

CPU side: readback copies contacts buffer alongside compact transform data in a single staging buffer. `processReadback` swaps the double-buffered `Contacts` resource (prev â†” current). `readContacts` iterates both buffers using a cursor (`ContactReader.lastTick`). Contacts persist for 2 physics ticks.

**`Contact.bodyA/bodyB` are GPU body indices, not ECS entity IDs.** Map via `physics.bodyEids[idx]` before doing any ECS work (`hasComponent`, `addComponent`, etc.). Cooldown maps keyed on the raw indices are fine â€” they're stable within a tick.

## Testing

`bun bench -- --scenario pile` â€” GPU Playwright benchmark. `--count 2000` for correctness, `--count 10000` for perf. Physics scenarios: `pile`, `physics`. Hull validation: `--scenario physics --test cone` (hull on ground), `--test mixed` (hull with all primitive types). Impulse validation: `--scenario physics --test impulse` (one-shot upward kick + angular impulse every 2s). Velocity validation: `--scenario physics --test velocity` (direct velocity override under 3x gravity every 2s).

**Test discipline:** stability checks must measure something concrete (energy conservation, drift, residual), not just "stays bounded." Validate GPU via readback, not eyeball. Fix failing tests instead of marking them `todo`. Tolerances derive from integrator order + step size, not observation. Compare against discrete-time references â€” continuous-time solutions diverge by an integration error term.

## Character

`Character` component â€” unopinionated GPU-driven kinematic AABB on a `Body` with mass=0. Accepts world-space velocity intent (`moveX`, `moveZ` in units/s) and a `jump` flag from a driver (e.g. Player). Executes GPU sweep, returns `grounded` state. No keyboard reading, no camera â€” those are driver concerns.

**Known limitation:** pushing a dynamic body against an immovable surface causes escalating constraint forces (CONSTRAINT_KINEMATIC alpha=0 + augmented Lagrangian lambda accumulation). The body eventually escapes violently. Correction feedback and pre-sweep approaches were tried and rejected â€” the fix requires solver-level changes (lambda clamping, warmstart suppression, or a dynamic character variant).

**Fields:** `speed` (base, reference for drivers), `maxSlope`, `jumpHeight`, `gravity` (character-specific, default 50), `mass` (character-controller concept, default 70 â€” solver never sees it; Body.mass stays 0 for kinematic physics), `grounded` (readback), `coyoteTime`, `moveX`, `moveZ`, `jump`.

**Dispatch order:** processReadback â†’ syncTransforms â†’ characterApply â†’ copy bodyâ†’prev â†’ LBVH â†’ broadphase â†’ narrowphase â†’ solver â†’ writeback â†’ characterSweep â†’ copy charâ†’prev. `characterApply` applies movement intent, adds ground body velocity (from previous frame's `charGroundIdx`), and sets `moved=1.0`. Character enters broadphase at intended position â€” solver generates CONSTRAINT_KINEMATIC for character-dynamic contacts. `characterSweep` (post-solver) handles depenetration, grounded detection, and writes `charGroundIdx` for next frame's velocity inheritance. processReadback skips kinematic bodies (mass â‰¤ 0) except characters â€” their Transform is CPU-owned. Write Transform directly for kinematic movement (platforms, levers, grab anchors). Characters are mass=0 but GPU-owned (sweep shader moves them), so readback writes their Transform.

**Body struct `moved` field** at offset 49 (offset 50 is `moveFlag`, 51 is padding). Set by `syncTransforms` only for kinematic bodies with the `Move` component (`moveFlag > 0.5`), when position changed from the previous frame (`dot(diff,diff) > 1e-10`). Without `Move`, syncTransforms zeros both `vel` and `moved` (teleport semantics). Also set by `characterApply` when character movement intent is non-zero. Read by narrowphase (`pushConstraintSearching`) to auto-promote CONSTRAINT_CONTACT to CONSTRAINT_KINEMATIC. BODY_STRIDE unchanged at 52.

**`Move` marker component** â€” opt-in for kinematic bodies whose displacement should impart velocity to contacts (platforms, elevators, conveyor belts). Without `Move`, kinematic position changes are treated as teleports (no friction drag, no ground velocity inheritance). `Move` flag is uploaded as the 8th section of the transform buffer and read by `syncTransforms`. Characters don't need `Move` â€” `characterApply` sets `vel`/`moved` directly.

**Coyote time:** `Character.coyoteTime` (default 0.1s). While grounded, timer resets to configured value. On leaving ground, timer counts down. Jump allowed while timer > 0 (consumed on use). Tracked per-character in `gpu.characterCoyoteTimers`.

**Readback:** Separate async readback for grounded flag (charData buffer). Position readback uses the existing compact readback (processReadback â†’ Transform). 1-2 frame delay. Player module interpolates camera between consecutive readback positions.

**Bind group:** Own layout (8 bindings: bodies, treeNodes, sortedBodyIds, leafAABBs, charData, charIndices, charParams, charGroundIdx). Separate from solver/broadphase layouts. Shader in `character.wgsl.ts`.

**Collision tests:** AABB-vs-OBB (6-axis SAT, face axes only), AABB-vs-Sphere, AABB-vs-Capsule. No hull support. Character is always axis-aligned (no rotation).

**Dynamic body push:** Handled by the solver via `CONSTRAINT_KINEMATIC` constraints (alpha=0 for moving kinematic contacts). Character sweep only does depenetration and grounded detection â€” no ad-hoc push.

**Player integration:** Add `Player` + `Character` to a body entity with a `Camera` + `Viewport` child entity (ChildOf relation). Player reads WASD/mouse, writes world-space moveX/moveZ/jump to Character, finds the camera child via ChildOf query, interpolates camera position at render rate with eye-height offset. Warns if no Camera child found. Scene pattern: `<a player character body ...><a camera viewport transform /></a>`. Gym scenario `?scenario=player` demonstrates this. 

## Debugging

- **Simple cases first:** hardcode 2-body scenarios, trace actual numbers. Contact bugs are usually geometry bugs
- **Readback:** `requestReadback` dispatches a compact shader (pos+quat only, 7 floats/body) then copies counters + compact buffer to staging. Async callback checks `bodyGeneration` â€” discards if bodies changed during flight. `processReadback` also guards on generation (prevents stale data writing to wrong entities when body list changed between callback and processing). Warns on constraint overflow, color overflow, hash overflow, stack overflow, adjacency overflow, uncolored bodies, and high hash occupancy. Syncs transforms to ECS
- **WGSL body copies:** `var body = bodies[idx]` copies. `bodies[idx].pos` still returns pre-solve until writeback
- **Penalty explosion:** check constraint sign, penalty growth rate, cached contact sentinel, lambda sign
